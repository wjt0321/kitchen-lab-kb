## 1. 产品概述

### 1.1 产品定位

兴达样品库知识库管理系统是一套**轻量、易迁移的本地样品库管理工具**，面向研发人员日常的样品主数据维护、配方试验记录和成功率分析需求。系统以"单文件数据库 + 桌面窗口"方式运行，无需联网、无需服务器、无需数据库安装，适合在 Windows 办公电脑上独立使用。

### 1.2 核心价值

| 痛点 | 解决方案 |
|:---|:---|
| 样品数据和配方记录散落在 Excel / 纸质笔记中，难以检索和复用 | 结构化存储，支持品号/品名模糊搜索、原料反查 |
| 同一配方多次试验，历史成功率难以统计 | 配方组合自动哈希，一键查看该组合的全部试验记录和成功率 |
| 换电脑后系统重装、环境配置繁琐 | 整个文件夹拷贝即用，离线依赖包内置于 `dependencies/` |
| 数据安全无保障 | 一键备份数据库为 ZIP，退出时自动备份，保留最近 5 份历史备份 |

### 1.3 产品边界

**本系统做**：
- 产品主数据的增删改查与归档
- 配方试验记录的录入、编辑、查询
- 配方组合的成功率自动统计
- Excel / JSON 数据导出
- 一键数据库备份与恢复

**本系统不做**：
- 多人协作与权限管理（当前为单用户版）
- 云端同步或远程访问（纯本地系统）
- 图片/附件管理
- 复杂的统计分析或图表

## 2. 目标用户与使用场景

### 2.1 目标用户

- **主要用户**：研发工程师 / 实验员，负责样品管理、配方试验和成功率跟踪
- **使用环境**：Windows 10/11 办公电脑，1024×768 及以上分辨率
- **技能要求**：基本的 Windows 操作能力，无需编程或数据库知识

### 2.2 典型使用场景

| 场景 | 描述 |
|:---|:---|
| **日常录入** | 做完一批试验后，打开系统录入产品编号、原料配比、试验结果 |
| **配方复盘** | 查看某产品下同一配方的历史试验记录，分析成功率变化趋势 |
| **原料反查** | 想知道"某个原料在哪些配方里用过"，通过配方列表按原料名称搜索 |
| **数据导出** | 月底导出产品列表和成功率汇总为 Excel，用于报告或分享 |
| **换电脑迁移** | 将整个项目文件夹拷贝到 U 盘，在新电脑上双击快捷方式即可启动 |

## 3. 核心功能需求

### 3.1 用户登录

- **需求**：用户输入姓名即可登录，无需密码
- **目的**：记录操作人，用于配方/产品的创建人追溯
- **规则**：登录名自动记录到产品/配方的 `created_by` 字段
- **登出**：退出时更新登出时间记录

### 3.2 产品管理

#### 3.2.1 产品列表
- 以表格形式展示全部活跃产品
- 列：品号、品名、规格、当前数量、状态、配方数、创建时间
- 支持关键词搜索（品号/品名模糊匹配）和状态下拉筛选
- 支持分页浏览

#### 3.2.2 新建产品
- 必填：品号（唯一）、品名、规格
- 选填：当前数量、备注
- 品号重复时后端返回错误提示

#### 3.2.3 产品详情
- 展示产品基本信息卡片
- 下方通过 Tab 切换「配方记录」和「成功率汇总」
- 配方记录支持按状态、原料名称、日期范围筛选
- 支持在详情页直接调整库存，记录变动流水

#### 3.2.4 编辑与归档
- 所有字段可编辑
- 归档为软删除（状态改为 `archived`），可从归档列表恢复
- 真删除需二次确认

### 3.3 配方管理

#### 3.3.1 新建配方
- 选择产品 → 填写试验日期、配方名称（可选）
- 动态添加原料/辅料行（类型、名称、用量、单位）
- 填写本次用量、试验状态（成功/失败/待观察）、备注
- 提交时系统自动计算配方组合哈希值

#### 3.3.2 配方列表
- 支持按产品、状态、日期范围筛选
- 支持按原料/辅料名称反查
- 录入时提供常用原料名称和单位的下拉建议

#### 3.3.3 配方详情
- 展示试验信息、原料列表、辅料列表
- 展示同组合（相同哈希）的全部试验记录及成功率
- 支持从已有配方快速复制生成一条待观察记录（保留原料辅料组合）
- 支持编辑和删除

### 3.4 成功率查询

#### 3.4.1 产品级成功率
- 在任一产品详情页的「成功率汇总」Tab 中查看
- 按配方组合（哈希）分组统计：试验次数、成功次数、成功率
- 成功率以数字百分比 + 颜色进度条双通道展示
- 支持最小试验次数筛选（如"只看试验≥3次的配方"）

#### 3.4.2 全局成功率
- 独立页面，跨产品浏览所有配方的成功率
- 支持按产品、日期范围、成功率区间、最少试验次数筛选
- 结果按成功率降序排列

### 3.5 数据导出

| 导出类型 | 格式 | 内容 |
|:---|:---|:---|
| 产品列表 | Excel (.xlsx) | 品号、品名、规格、数量、状态、创建时间 |
| 配方记录 | Excel (.xlsx) | 产品、日期、配方名称、状态、用量、备注 |
| 成功率 | Excel (.xlsx) | 产品、配方组合、试验次数、成功次数、成功率 |
| 导入模板 | Excel (.xlsx) | 含表头的空模板，用于批量导入 |
| JSON 全量 | JSON | 全部产品 + 配方 + 原料辅料，用于跨系统迁移 |

所有导出文件集中存放在 `exports/` 目录，`_latest/` 子目录保存每类的最新文件。

### 3.6 导入

- 支持导入本系统导出的 JSON 文件或备份 ZIP 包
- 导入会**替换**当前数据库（非追加），系统在导入前自动备份
- 适用于数据迁移或从备份恢复

### 3.7 备份与恢复

- 一键将当前数据库 (SQLite) 打包为 ZIP 存入 `backups/database/`
- 数据库备份保留最近 5 份，新的自动覆盖最旧的
- 退出系统时会弹窗确认并自动执行一次备份
- 备份失败时给出明确错误提示（如磁盘空间不足、目录权限问题）

### 3.8 库存调整

- 在产品详情页可调整当前数量
- 每次调整记录变动前后数值、变动原因和操作人
- 支持查看该产品的全部库存变动流水

## 4. 数据模型

### 4.1 核心实体

| 实体 | 说明 | 核心字段 |
|:---|:---|:---|
| **产品 (Product)** | 样品主数据 | 品号(唯一)、品名、规格、当前数量、状态(active/archived) |
| **配方 (Recipe)** | 每次试验一条记录 | 产品ID、试验日期、配方名称、配方哈希、状态(success/failed/pending)、用量 |
| **原料辅料 (Material)** | 配方中的用料明细 | 配方ID、类型(原料/辅料)、名称、用量、单位 |
| **库存流水 (InventoryMovement)** | 库存变动记录 | 产品ID、变动前、变动后、变动原因、操作人 |
| **用户登录 (UserLogin)** | 登录/登出记录 | 用户名、登录时间、登出时间 |

### 4.2 关键算法：配方组合哈希

同一产品的"相同原料+相同辅料+相同用量"视为同一配方组合。系统在新建配方时自动计算 SHA-256 哈希值：

1. 原料列表按名称排序 → 辅料列表按名称排序
2. 组合为 `{品号, 原料[{名称,用量,单位}], 辅料[...]}` 的 JSON
3. 计算 SHA-256，取前 16 位作为配方哈希

**效果**：改一个原料用量 → 哈希变化 → 识别为新配方；完全相同的原料辅料组合 → 哈希相同 → 归为同一配方，可统计成功率。

## 6. 非功能性需求

### 6.1 性能

| 指标 | 目标 |
|:---|:---|
| 产品列表加载 | <1s（1,000 条产品） |
| 配方列表加载 | <2s（10,000 条配方，含筛选） |
| 成功率计算 | <1s（单产品全部配方组合） |
| Excel 导出 | <3s（全量数据） |
| 启动时间 | <10s（含依赖检查） |

### 6.2 可靠性

- 数据库操作使用 SQLite 事务，保证原子性
- 导入前自动备份，避免误操作数据丢失
- 退出流程确保先备份再停止进程
- 备份失败时有明确错误提示，不会静默跳过

### 6.3 安全性

- 纯本地运行，不连公网，不开放外部端口
- 用户名仅用于操作记录，不做身份认证
- `.env`、`data/`、`backups/` 目录不纳入 Git 版本管理
- 导入文件格式校验，拒绝非本系统导出的文件

### 6.4 易用性

- 双击快捷方式即可启动，无需命令行操作
- 首次启动自动从 `dependencies/` 离线安装依赖
- 表单校验行内提示，不弹窗打断录入流程
- 空状态页面提供明确引导（如"暂无产品，点击新建"）
- 退出时自动备份，降低数据丢失风险

### 6.5 可迁移性

- 整个项目文件夹复制到新电脑即可运行
- 离线依赖包内置，目标电脑无需联网
- 数据库文件随文件夹迁移，无需导出/导入
- 面向 Windows + Python 3.14 x64 环境优化

### 6.6 兼容性

| 维度 | 要求 |
|:---|:---|
| 操作系统 | Windows 10 / 11 |
| Python | 3.14 x64（推荐） |
| 屏幕分辨率 | ≥1024×768（推荐 1100×760） |
| 浏览器 | Edge WebView2（Windows 内置） |

## 7. 部署与迁移

### 7.1 首次部署

1. 确保电脑已安装 Python 3.14（命令行可执行 `python --version`）
2. 将 `kitchen-lab-kb/` 文件夹复制到目标电脑
3. 双击 `兴达样品库知识库.lnk` 或 `启动样品库知识库.bat`
4. 首次启动会自动检查并安装依赖（从 `dependencies/` 离线安装）
5. 浏览器自动打开 `http://127.0.0.1:7777/`，输入用户名即可登录

### 7.2 换电脑迁移

1. 在旧电脑点击右上角「退出」，让系统自动备份并停止进程
2. 复制整个 `kitchen-lab-kb/` 文件夹到 U 盘或新电脑
3. 在新电脑按"首次部署"步骤操作
4. 登录后检查产品列表、导出和备份是否正常

### 7.3 运行数据说明

以下目录/文件为运行时生成，迁移时建议一并复制：
- `data/kitchen.db` — 主数据库
- `backups/database/` — 历史备份
- `exports/` — 导出文件
- `logs/` — 运行日志

# Kitchen Lab UI Redesign Design

## 背景

当前 `kitchen-lab-kb` 已完成 MVP，可满足本地样品库知识库的核心使用需求，但前端界面整体仍偏传统后台风格，视觉质感与交互体验不足。目标是在不改变核心架构、后端接口、数据库结构和主要业务流程的前提下，将现有前端升级为更接近参考图风格的桌面工作台界面。

本次设计聚焦以下前端文件与交互层：

- `templates/base.html`
- `static/style.css`
- `static/app.js`

业务边界保持不变：

- 产品
- 配方
- 库存流水
- 成功率查询
- 导入导出
- 备份
- 退出

## 目标

### 主要目标

- 高保真借鉴参考图的整体气质、层级、留白和桌面工作台感。
- 保留现有 hash 路由、接口调用方式与主要数据模型。
- 明显提升登录、产品检索、产品管理、配方录入和成功率查询的可用性。
- 把当前“能用的管理页”升级为“桌面业务工作台”。

### 非目标

- 不新增新的业务模块。
- 不修改后端接口契约。
- 不改数据库结构。
- 不引入新的前端框架。
- 不把应用改造成聊天产品或完全不同的信息架构。

## 设计原则

### 1. 外壳优先

优先重构全局工作台外壳，再把现有业务页面映射进去，而不是只做颜色和圆角层面的换皮。

### 2. 架构不动，表达重构

不改业务对象和核心流程，但重新组织页面中的视觉层级、操作位置和信息优先级。

### 3. 强化桌面工作台感

重点突出左侧项目栏、顶部轻工具栏、中间主工作区和柔和悬浮面板，让界面更接近参考图。

### 4. 常用页面强复刻，复杂页面同语言适配

登录页、全局工作台框架、产品列表页作为风格样板页，优先实现较强的参考图气质。产品详情、配方详情、成功率页在同一设计语言下保持业务清晰度。

### 5. 视觉和效率并重

界面密度比当前略松，但不能影响录入效率。最终效果应是“高级但能干活”。

## 目标体验

### 整体气质

- 浅灰白、低饱和、克制、安静。
- 使用超轻边框、柔和玻璃感、较强留白和清晰层级。
- 减少传统后台的重按钮、重卡片、重表格感。

### 信息层级

页面固定为三层：

1. 全局工作台外壳
2. 当前页面主任务区
3. 辅助信息区

同一屏不允许多个同权重区域同时争抢注意力。

### 操作优先级

- 每页只保留一个最强主动作。
- 常用操作前置。
- 危险操作后置，并在视觉上明显降噪或隔离。

## 信息架构

### 全局框架

当前结构为“顶部导航 + 内容区”，重构后改为：

- 左侧工作台栏
- 顶部轻工具栏
- 中央主工作区

其中：

- 左侧栏负责全局导航和项目感知。
- 顶部栏负责用户、备份、导入、导出、退出等系统操作。
- 中央区只承载当前业务页面内容。

### 页面映射

#### 登录页

- 保留用户名直接登录，不新增密码逻辑。
- 使用大留白中心区域。
- 输入框与主按钮成为主焦点。
- 保留上次登录人提示，但弱化其视觉权重。

#### 产品列表

- 作为默认主工作台首页。
- 顶部改为“主任务区”，承载查找产品、新建产品和状态切换。
- 下方为产品明细区。
- 空状态根据“无数据”和“筛选无结果”分别呈现。

#### 产品详情

- 顶部聚焦产品身份信息与高频动作。
- 将“新建配方”“调整库存”提升为主工作区操作。
- 保留库存流水与 tab 切换，但降低其与页面头部的竞争关系。

#### 配方详情

- 强化配方基本信息、当前状态和同组合历史的阅读顺序。
- 保留原料、辅料分组展示。
- 保留复制、编辑、删除动作，但按主次重新组织。

#### 成功率页

- 继续作为独立页面保留。
- 筛选区按主次重新组织。
- 成功率、试验次数和查看入口成为视觉重点。
- hash 信息保留，但展示方式更人类可读。

#### 系统级操作

- 备份、导入、导出、退出统一放到顶部系统区。
- 导出保留菜单式入口。
- 危险动作和普通动作拉开视觉距离。

## 视觉语言

### 色彩

- 主背景：浅灰白和暖白渐变。
- 组件底：半透明浅色面板。
- 边框：低对比、细边框。
- 主强调色：近黑或深灰，用于主按钮和标题聚焦。
- 辅助强调色：少量低饱和青绿，用于链接、状态和局部提示。
- 危险色：克制使用，仅用于删除、退出等不可逆操作。

### 形态

- 大圆角但不过分卡通。
- 面板和菜单采用轻玻璃感。
- 阴影柔和，强调悬浮关系而非强烈投影。

### 字体和排版

- 中文优先清晰易读。
- 标题比当前更有层次，但不夸张。
- 正文信息偏轻，辅助说明进一步降权。
- 控制整体字重和留白，使界面更安静。

### 控件规则

- 主按钮：深色高对比、圆角更完整。
- 次按钮：浅底弱边框。
- 文本按钮：只用于辅助操作。
- 标签和 badge：弱化装饰性，强调信息状态。

## 组件规范

### 左侧工作台栏

职责：

- 展示品牌与项目名称
- 承载主导航
- 可选展示最近操作或辅助入口

规则：

- 风格轻，不使用传统深色管理后台菜单样式。
- 当前项通过背景块、字重和微弱状态线突出。
- 不堆叠过多层级。

### 顶部轻工具栏

职责：

- 展示当前用户
- 承载备份、导入、导出、退出

规则：

- 从主导航角色降级为系统操作栏。
- 按钮减少“全部都很重”的问题。
- 导出使用更自然的下拉菜单。

### 页面主任务区

每个核心页面顶部都应有主任务区，负责承载当前页面最重要的动作和说明：

- 产品列表：搜索、筛选、新建
- 产品详情：查看概览、调整库存、新建配方
- 成功率页：筛选结果、定位值得看的组合

要求：

- 比普通卡片更大
- 留白更多
- 更贴近参考图中央输入区的气质

### 表格

- 保留表格作为主要承载方式。
- 降低 Excel 感，提升工作台明细面板感。
- 表头更轻，行高更舒展。
- hover 更细腻。
- 操作入口减少视觉噪音。
- 针对桌面端优化，不以移动端优先。

### 表单

- 保留现有字段与保存逻辑。
- 从“左右标签 + 输入框”的传统录入样式升级为更清晰的分组式输入区。
- 试验信息、原料辅料、结果状态形成明确三段式。
- 保存按钮固定为当前页面唯一主动作。

### 弹层、提示和空状态

- 统一 modal、toast、帮助说明、空状态语言。
- 成功提示更轻。
- 失败提示更聚焦问题。
- 空状态必须带有下一步操作。

## 交互设计

### 登录

- 页面加载后自动聚焦用户名输入框。
- 支持回车登录。
- 登录成功直接进入产品工作台。

### 产品列表

- 搜索框成为第一视觉焦点。
- 状态筛选、重置和新建围绕搜索任务组织。
- 行点击与查看、编辑关系重新梳理，减少多重可点击区域带来的歧义。
- 明确区分“无产品数据”和“筛选无结果”。

### 产品详情

- 头部优先展示产品身份和关键动作。
- “新建配方”“调整库存”更靠前。
- tab 保留，但视觉上更接近工作区切换。

### 库存调整

当前为浏览器 `prompt`。设计中改为站内轻量弹层，接口不变。

弹层要求：

- 同时录入变动数量和原因
- 错误提示更明确
- 不引入新的后端依赖

### 配方录入

- 原料 / 辅料区增强新增和删除的反馈感。
- 常见错误集中提示。
- 保持当前保存逻辑与校验逻辑。

### 成功率查询

- 筛选项按常用与进阶组织。
- 强化成功率、试验次数和查看入口。
- hash 继续保留，但展示更收敛。
- 展开历史更自然，像工作区内联明细。

### 系统反馈

- 加载态比当前更明确。
- 成功反馈短促明确。
- 失败反馈避免只有“失败”二字，应尽量呈现下一步判断信息。

## 实现范围

### 允许改动

- 重写 `templates/base.html` 的全局布局
- 大幅调整 `static/style.css`
- 重构 `static/app.js` 的前端渲染结构
- 将原生 `prompt` 替换为站内弹层

### 不允许改动

- 后端 API 契约
- 数据库结构
- 核心路由语义
- 业务对象定义

## 实施顺序

### 阶段 1

- 全局框架
- 左侧栏
- 顶部系统区
- 登录页

### 阶段 2

- 产品列表页，作为风格样板页

### 阶段 3

- 产品详情页
- 配方录入页

### 阶段 4

- 配方详情页
- 成功率查询页

### 阶段 5

- modal
- toast
- 空状态
- 菜单
- 统一反馈组件

## 风险与缓解

### 风险 1：结构和样式耦合

当前前端使用字符串拼接渲染，页面结构与样式高度耦合。大改时容易牵一发动全身。

缓解方式：

- 先建立统一页面壳和设计 token
- 再逐页迁移渲染结构

### 风险 2：单文件复杂度过高

`static/app.js` 承担全部页面渲染和交互，改造过程中容易互相影响。

缓解方式：

- 先按页面职责重组渲染模板和辅助方法
- 每完成一个核心页就做可视和功能回归

### 风险 3：只有表面像，没有整体高级感

如果只做样式覆盖而不重整结构，最终只能得到“更好看一点”的后台页面。

缓解方式：

- 先完成全局外壳和工作台布局
- 再做组件与页面内层细节

## 验收标准

### 功能验收

以下流程必须保持可用：

- 登录
- 产品搜索
- 新建产品
- 编辑产品
- 调整库存
- 录入配方
- 复制配方
- 成功率筛选
- 导入
- 导出
- 备份
- 退出

### 体验验收

- 桌面端主视图具有明显参考图风格气质。
- 页面层级比当前更清晰。
- 高危操作不易误触。
- 高频操作更容易发现。
- 表格和表单既更美观，也保持可录入性。

### 兼容性验收

- 桌面端为本次主优化目标。
- 窄屏不崩，但不追求与桌面端同等表现。

## 结论

本次改造不是简单换皮，而是在保留现有业务与技术边界的前提下，重做前端工作台外壳、页面层级和关键交互，使 `kitchen-lab-kb` 从 MVP 管理页升级为具有参考图风格的桌面业务工作台。
